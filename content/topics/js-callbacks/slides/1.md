## JavaScript Callback Functions

---

## What is a callback?

> A function that another function invokes as part of its procedure

**You don't call the callback**!

A callback function is executed by a **procedure function**.

```js
// receives the callback as an argument
const procedureFunction(callbackFunction) {
  ...
  // execute the callback as part of its procedure
  callbackFunction()
  ...
}
```

___

### Higher Order Functions (HOF)

> The callback pattern is an example of a HOF

A HOF is a function which meets one or both of:

- receives a function as an argument
- returns a function

Procedure functions are *higher order* because they **receive the callback as an argument**.

___

### Functional JavaScript

> The functional paradigm of JavaScript enables callbacks

Recall that in JavaScript functions are first class:

- they can be defined anywhere
- they can be passed as arguments to a function
- they can be returned from a function

___

> They are just functions!

The fundamental concepts of a function still apply to both the **callback** and **procedure**:

- What inputs (arguments) are received?
- What logic needs to be performed inside?
- What output is returned?

## Responsibilities

> *Typically* you use but do not write the procedure itself

The procedure function abstracts tedious or complex logic for you.

You will only be responsible for:

- writing the callback
- calling the procedure and providing it the callback as an argument

## Procedure Functions

> A function which receives the callback as an argument

Procedure functions are designed to abstract away procedures that are complex or tedious

___

> A loop procedure

```js
const count = 3;
const printIndexCallback = (index) => console.log(index);
for(let i = 0; i < count; ++i) {
  printIndexCallback(i);
}
```
