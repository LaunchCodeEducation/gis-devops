- definition
  - a function passed as an argument to another function (outer)
  - it is invoked at a later time by the procedure not by you
- HoF
  - JS functions are first class citizens
    - functions can be defined anywhere, passed as args, returned from other functions
  - a function that takes another function as an argument
  - and/or a function that returns a function

GET TO AN EXAMPLE ASAP
- plain english example

- key points
  - abstracts a procedure that is complex or tedious
  - what your callback function handles:
    - what params are used
    - what is returned
  - what the procedure function handles:
    - when the callback is executed
    - what arguments are given to the callback
    - what effect the callback return value has on the procedure
  - callbacks are executed by another function
  - callback signatures are defined by the procedure function
    - this includes the ordered params and the return value usage
    - find in documentation
- your steps
  1. find the signature of the callback
  2. determine which (if any) params you will use (ignores other)
  3. determine what (if any) value is returned
  4. providing the callback to the procedure function (in-line or external)
- defining in-line vs external
  - in-line: anonymous func as arg
  - external: callback reference as arg
- usage in js
  - iterative methods (string, array)
  - DOM events
  - node events
  - promises
