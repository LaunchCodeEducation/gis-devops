## Iterative Array Methods

---

## What are they?

> `Array` methods that simplify common iterative tasks

Each of these methods are `procedure` functions:

- `forEach()`: iteration over each element
- `filter()`: iteration to remove certain elements
- `map()`: iteration to transform each element
- `sort()`: iteration to sort the elements

___

### How are they used?

While each procedure differs in behavior they all:

- iterate over the source `Array`
- receive a single `callback` argument
- invoke the `callback` on each iteration

```js
[].forEach(callback)
[].filter(callback)
[].map(callback)
[].sort(callback)
```

___

### In-line & External

> You can define an in-line or external `callback`

```js
// externally defined
const callback = () => { ... }
[].forEach(callback)
// in-line
[].forEach(() => { ... })
```

___

### Chaining

> If the method **returns an `Array`**

You may chain multiple methods onto each other

```js
// filters then transforms the elements
[].filter(filterCallback).map(transformCallback)
```

Notes:

stress that the same rules of method chaining apply (output must match next input)

---

## Element, Index, Array

> The most common `callback` parameters

`forEach()`, `map()` and `filter()` callbacks all receive the following positional arguments:

1. `element` (`e`): the current `element` in the iteration
2. `index` (`i`): the current `index` in the iteration
3. `array` (`a`): the `Array` being iterated over

Notes:

the shorthand is often used

___

> Not all arguments need to be used

JavaScript allows you to ignore a positional argument if it will not be used

```js
const callback = () => {} // ignores the element, index and array
const callback = (e) => {} // ignores the index and array
const callback = (e, i) => {} // ignore the array
const callback = (e, i, a) => {} // utilizes all three arguments
```

___

> Always name the `element` according to the use case

The arguments are *positional* so you may rename them in your `callback`

```js
[1, 2, 3].forEach((number) => { ... })

["vamp", "paul"].forEach((name) => { ... })

[
  { id: 1, text: "buy a costume"},
  { id: 2, text: "buy candy"},
].forEach((todoItem) => { ... })
```

## `forEach()`

> Used to iterate over each element

```js
["one", "two", "three"].forEach((element, index, array) => {
  console.log(element); // "one", ...
  console.log(index); // 0, ...
  console.log(array); // ["one", "two", "three"]
})
```

___

> method signature

```js
forEach(callback) -> void
```

> callback signature

```js
(element, index, array) -> void
```

___

### When to use it

> When you need to **read the contents** of an `Array`

`forEach()` simply loops over an `Array` and invokes the `callback` on each iteration, it does not:

- return anything
- use the output of the `callback`

Notes:

the `forEach()` method is not commonly used but is a simple introduction because it shares the same callback args as map and filter
