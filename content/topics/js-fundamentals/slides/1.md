## JavaScript Introduction

---

### JavaScript

> A programming language that was originally created to allow for more complex and interactive websites.

JavaScript's original environment was a web browser since it was responsible for client side scripting needs of websites.

---

### Node.js

JavaScript has progressed from its original objective of handling client side scripting.

> `Node.js` provides the ability to run JavaScript outside of a browser.

___

#### Node.js Uses

`Node.js` can be used for many things including:

- scripting
- creating web servers
- creating services
- creating CLI applications

---

### EcmaScript

> EcmaScript is the JavaScript standard containing the information necessary to implement JavaScript.

JavaScript that is executed in a browser, and `Node.js` are both implementations of EcmaScript and therefore vary slightly.

___

#### EcmaScript Versions

> EcmaScript had a major revision come out in 2015. It is known as EcmaScript 2015 (ES2015) or EcmaScript 6 (ES6).

Since then a yearly version of EcmaScript has been released with minor changes and improvements.

___

##### EcmaScript Information

Checkout [Ecma International](https://www.ecma-international.org/publications-and-standards/standards/ecma-262/) for more information about EcmaScript and its various versions.

---

### Interpreted

> JavaScript is an interpreted programming language.

---

### Dynamically Typed

> JavaScript is a dynamically typed programming language.

---

### Common Paradigms

JavaScript is a multi-paradigm language and you will regularly see aspects from both the Functional and the Object-Oriented paradigms.
