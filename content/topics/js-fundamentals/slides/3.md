## Programming Fundamentals

### Arrays & Object Literals

---

## Arrays

> Arrays are JavaScript's list-like objects.

Arrays allow you to store **any type** of elements.

Arrays are indexed starting at 0 and their size is indefinite.

___

### Array Examples

> Array elements can be mixed

```js
const someArray = ["a", 5, false];
```

> Indexing is performed using bracket notation

```js
const someArray = ["a", 5, false];
console.log(someArray[0]); // prints "a"
console.log(someArray[1]); // prints 5
console.log(someArray[2]); // prints false
```

___

### Array `length` Property

Arrays have a built in `length` property that will return the number of elements in the array.

```js
const someArray = ["a", 5, false];
console.log(someArray.length); // 3
```

___

### Sub-Arrays

> An array can hold a **nested** array as an element.

Indexing is performed using left-to-right bracket notation:

```js
const someArray = ["a", 5, false, [10, 9, 8]];
console.log(someArray[3]); // prints [10, 9, 8]
console.log(someArray[3][0]); // prints 10
console.log(someArray[3][1]); // prints 9
console.log(someArray[3][2]); // prints 8
```

---

## Non Iterative Array methods

> Array methods can be iterative or non-iterative

- `.indexOf()`
- `.slice()`

___

### `.indexOf()`

> Searches the array for a value, and returns the index of the first occurrence.
 
If the value is not found it returns `-1`.

```js
const someArray = ["a", "b", "c"];
console.log(someArray.indexOf("a")); // prints 0
console.log(someArray.indexOf("c")); // prints 2
console.log(someArray.indexOf("dog")); // prints -1
```

___

### `.slice()`

> Takes an array and returns a new array starting at a specific index.

A smaller *slice* of elements is produced:

```js
const someArray = ["a", "b", "c"];
console.log(someArray.slice(0)); // prints ["a", "b", "c"]
console.log(someArray.slice(1)); // prints ["b", "c"]
console.log(someArray.slice(2)); // prints ["c"]
console.log(someArray.slice(1, 2)); // prints ["b"]
```

## Object Literal

> Object literals are JavaScript's key-value data structures.

They are the *map* data structure and should not be confused with generic `Object` types

___

### Object Literal Example

> Object literals are created using curly braces

Any key that follows **valid variable naming**:

- can be added without quotes
- accessed or changed via dot-notation

```js
const dog = { name: "Bernie", age: 5 };
console.log(dog.name); // prints "Bernie"
dog.age = 12; // reassign
console.log(dog.age); // prints 12
dog.color = "brown"; // added to object
```

___

> Non-standard keys

Keys that do not follow valid JavaScript variable rules:

- added as a string using quotes
- accessed or changed via bracket notation

```js
const myObject = { "key with spaces": "value" };
console.log(myObject["key with spaces"]) // "value"
```

___

### Object Static Methods

> Object literals are simple data structures without methods of their own.

You can work with them using the standard `Object` static methods:

- `keys()`: returns an array of just the keys
- `values()`: returns an array of just the values
- `entries()`: returns a 2-D array of key-value pairs

```js
const dog = { name: "Bernie", age: 5};
console.log(Object.keys(dog)); // ["name", "age"]
console.log(Object.values(dog)); // ["Bernie", 5]
console.log(Object.entries(dog)); // [["name", "Bernie"], ["age", 5]]
```
