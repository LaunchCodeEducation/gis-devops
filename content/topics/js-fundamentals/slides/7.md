### Type Coercion

> Type coercion is the **implicit conversion** between data types.

As a developer you may manually convert between data types, known simply as *type conversion*:

```js
const numberAsString = "10.5"
const number = parseFloat("10.5");
```

In JavaScript sometimes the context requires a type conversion that is initiated by JavaScript and not manually by the developer. When this happens we refer to it as **type coercion**.
___

> type coercion example:

<!-- CLARITY: clarify that both values are only changed temporarily/internally for comparison and does not effect the actual value / variable
-->

```js
const someString = "Hello";
if (someString) {
  console.log(someString);
}
```

This is valid code in JavaScript. The variable someString with the value "Hello" must resolve to either true or false so the conditional statement can determine if it should execute or not. In this case the value "Hello" is considered a truthy value and so is converted to true.

### Truthy & Falsy

> Due to type coercion **all** values must be resolved to a boolean representation of true or false when a boolean condition is encountered.

If a value resolves to true it is known as **truthy**. Most values are considered truthy.

If a value resolves to false it is known as **falsy**. The following slide contains all the falsy values.

### Strict vs Loose Equality

<!-- DISCUSS: consider moving this into the conditionals slides

if moved leave the inner workings of type coercion in this slide deck and just put the practical usage explanations over there
-->

Recall the equality operators:

- `==`: equality
- `!=`: inequality
- `===`: identity
- `!==`: nonidentity

The difference between equality and identity comparisons is type coercion.

___

equality operators:

When using the equality (`==`) and inequality (`!=`) operators the two values are implicitly converted to the same type first and then the values are compared.

<!-- CLARITY: "an attempt is made to coerce"

errors can be thrown if one type cannot be converted to the other (this is a common gotcha that is tough to debug since it is implicit)
-->

identity operators:

When using the identity (`===`) and nonidentity (`!==`) operators the two types are compared first and if they differ it returns false before performing a value comparison.

___

#### Loose vs Strict Equality Examples

loose equality:

```js
5 == 5 // no type conversion necessary and values are the same: return true
5 == "5" // convert types to the same and compare values: return true
```

strict equality:

```js
5 === 5 // same types, compare values return true
5 === "5" // different types return false
```
