### Primitives

- `Boolean`: true, false
- `Number`: integers, floating point numbers
- `String`: letters, numbers, special characters

___

JavaScript has a couple of additional primitive data types, but they are outside the scope of this class.

- `BigInt`
- `Symbol`

### Objects

- `Array`: array or list
- `Object Literal`: key value pair
- `null`: special type of object

___

JavaScript has quite a few more built in Object data types, but they are outside the scope of this class.

### Variables

- declaration: initialize a new variable
- assignment: set a variable's value
- `undefined`: declared but unassigned variable (`undefined` is a primitive data type!)
- mutability: a variable can be mutable (value can be changed), or immutable (value cannot be changed)

---

### Mutable Variables

> The keyword `let` allows us to declare a new mutable variable.

___

#### Mutable Declaration

```js
let x; // declares a mutable variable name: x, value: `undefined`
let y; // declares a mutable variable name: y, value: `undefined`
let z; // declares a mutable variable name: z, value: `undefined`
```

___

#### Mutable Assignment

```js
x = false; // assigns the variable named x to the value of 5
y = 5; // assigns the variable named y to value of false
z = "hello"; // assigns the variable named z to the value of "hello"
```

___

#### Declaration & Assignment

Many times you will declare a variable and assign it's value in one statement:

```js
let x = false; // x declaration and assignment to value false
let y = 5; // y declaration and assignment to value 5
let z = "hello"; // z declaration and assignment to value "hello"
```

---

### Immutable Variables

> The keyword `const` allows us to declare a new immutable variable.

```js
const x = 15;
```

When using the `const` keyword to declare an immutable variable **you must assign** the variable a value!

___

After declaring an immutable variable the value of the variable is fixed, meaning you cannot change or reassign the variable's value.

```js
const x = 15; // immutable variable declaration & assignment
x = 16; // not allowed
x += 1; // not allowed
console.log(x + 1); // allowed
```

---

### Block Scoping

> A variable's scope is determined by where the variable is declared.

___

```js
{ // scope 1
  // x does not exist here because it has not been declared
  let x; // declare variable in scope 1
  { // scope 2
    x = 5; // x exists here
  } // end scope 2
  console.log(x); // x exists here
} // end scope 1
console.log(x); // x DOES NOT exist here
```

`x` is declared in scope 1 and is usuable in scopes 1 and 2 after it has been declared.

However, `x` is not in scope outside of scopes 1 and 2, or before x has been declared.

### Array

> Arrays are JavaScript's list-like objects.

Arrays allow you to store any number of primitive literal values, variables, or objects as the **elements** of that array.

Arrays are indexed starting at 0.

___

#### Array Examples

```js
const someArray = ["a", 5, false];
```

someArray elements:

- 0: "a"
- 1: 5
- 2: false

```js
const someArray = ["a", 5, false];
console.log(someArray[0]); // prints "a"
console.log(someArray[1]); // prints 5
console.log(someArray[2]); // prints false
```

___

#### Array `length` Property

Arrays have a built in `length` property that will return the number of elements in the array.

```js
const someArray = ["a", 5, false];
console.log(someArray.length); // prints 3, the number of elements in someArray
```

___

#### Subarrays

An array can hold a different array as an element.

```js
const someArray = ["a", 5, false, [10, 9, 8]];
console.log(someArray[3]); // prints [10, 9, 8]
console.log(someArray[3][0]); // prints 10
console.log(someArray[3][1]); // prints 9
console.log(someArray[3][2]); // prints 8
```

### Conditional Logic

- equality operators
- relational operators
- logical operators

___

#### Equality Operators

- `==`: equality
- `!=`: inequality
- `===`: identity
- `!==`: nonidentity

___

equality & inequality examples:

```js
5 == 5 // true
5 == "5" // true
5 != 5 // false
5 != "5" // false
```

identity & nonidentity examples:

```js
5 === 5 // true
5 === "5" // false
5 !== 5 // false
5 !== "5" // false
```

___

#### Relational Operators

- `>`: greater than
- `<`: less than
- `>=`: greater than or equal to
- `<=`: less than or equal to

___

greater than & less than examples:

```js
5 > 5 // false
5 < 5 // false
5 > 4 // true
4 < 5 // true
```

greater than or equal to & less than or equal to examples:

```js
5 >= 5 // true
5 <= 5 // true
5 >= 4 // true
4 <= 5 // true
```

___

#### Logical Operators

- `&&`: and
- `||`: or

___

logical and examples:

```js
(5 > 8) && (5 > 6) // (false) && (false) -> false
(5 > 4) && (5 > 5) // (true) && (false) -> false
(5 > 9) && (5 > 4) // (false) && (true) -> false
(5 > 3) && (5 > 4) // (true) && (true) -> true
```

logical or examples:

```js
(5 > 8) || (5 > 6) // (false) || (false) -> false
(5 > 4) || (5 > 5) // (true) || (false) -> true
(5 > 9) || (5 > 4) // (false) || (true) -> true
(5 > 3) || (5 > 4) // (true) || (true) -> true
```

### Conditional Statements

- `if`
- `else`
- `else if`

___

#### `if`

```js
if (x > 5) { // if the value of x is greater than 5
  // do something
}
```

The code inside the if block is only executed if the condition is satisfied.

___

#### `else`

```js
if (x > 5) { // if the value of x is greater than 5
  // do something
}
else { // if the original if statement was not true
  // do something
}
```

If the original `if` statement is not satisfied the code inside the else block will be executed.

One of the branches **must** execute.

___

#### `else if`

```js
if (x > 5) { // if the value of x is greater than 5

}
else if (x < 2) { // if the value of x is less than 2 and the preceding if statement was false

}
```

If the original `if` statement is not satisfied the conditional statement in the following `else if` statement resolves.

If the `else if` condition is satisfied the `else if` block of code executes.

___

#### Chaining `else if` statements

You can add any number of additional `else if` statements to an existing `if` statement.

```js
if (a) { // if a resolves to true

}
else if (b) { // if b resolves to true and the preceding if statement was false

}
else if (c) { // if c resolves to true and the preceding if statement was false

}
```

`else if` chains always resolve in order, and only one code block can execute.

___

#### Extending `else if` with `else`

An `else` statement can be added after any number of `else if` statements.

```js
if (a) { // if a resolves to true

}
else if (b) { // if b resolves to true and the preceding if statement was false

}
else if (c) { // if c resolves to true and the preceding if statement was false

}
else { // if ALL preceding if statements were false

}
```

### Loops

---

### `while` loops

> Using the keyword `while` a code block will be executed until some condition is met.

Used when you do not know how many times a code block will need to be executed.

___

#### `while` example

```js
let x = 0;
while(x !== 5) {
  // code to be executed
  // x undergoes some change
}
```

---

### `for` loops

> Using the keyword `for` a code block will be executed a specific amount of times.

Used when you do know how many times a code block needs to be executed.

___

#### `for` example

```js
for (let x = 0; x < 100; x++) {
  // code to be executed exactly 100 times (0-99)
}
```

___

#### `for of`

> Code to be executed for each iterable in an **iterable object**.

`Strings` and `Arrays` are two examples of iterable objects.

___

##### `for of` examples

String example:

```js
const greeting = "hello";
for (const letter of greeting) {
  // code to be executed 
  // the variable: letter is available in this scope
  console.log(letter); // on iteration 1 this would print "h";
}
```

Array example:

```js
const names = ["Bernie", "Zelda", "Dutch"];
for (const name of names) {
  // code to be executed
  // the variable: name is available in this scope
  console.log(name); // on iteration 1 this would print "Bernie"
}
```

___

#### `for in`

> Code to be executed for each **enumerable property** of an object.

___

##### `for in` example

```js
const testObject = {"a": 15, "b": false, "c": "hello"};
for (const property in testObject) {
  // code to be executed
  // the variable: property is available in this scope
  console.log(property); // this would print one of "a", "b", or "c" on each iteration
  console.log(testObject.get(property)); // this would print one of 15, false, or "hello" on each iteration
}
```

JavaScript Objects have no memory of the order in which keys value pairs were inserted into the object.

This means you cannot predict the order of properties that will be looped through when using a `for in` loop.
