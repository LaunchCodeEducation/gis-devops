### Variables

- declaration: initialize a new variable
- assignment: set a variable's value
- `undefined`: declared but unassigned variable (`undefined` is a primitive data type!)
- mutability: a variable can be mutable (value can be changed), or immutable (value cannot be changed)

---

### Mutable Variables

> The keyword `let` allows us to declare a new mutable variable.

___

#### Mutable Declaration

```js
let x; // declares a mutable variable name: x, value: `undefined`
let y; // declares a mutable variable name: y, value: `undefined`
let z; // declares a mutable variable name: z, value: `undefined`
```

___

#### Mutable Assignment

```js
x = false; // assigns the variable named x to the value of 5
y = 5; // assigns the variable named y to value of false
z = "hello"; // assigns the variable named z to the value of "hello"
```

___

#### Declaration & Assignment

Many times you will declare a variable and assign it's value in one statement:

```js
let x = false; // x declaration and assignment to value false
let y = 5; // y declaration and assignment to value 5
let z = "hello"; // z declaration and assignment to value "hello"
```

---

### Immutable Variables

> The keyword `const` allows us to declare a new immutable variable.

```js
const x = 15;
```

When using the `const` keyword to declare an immutable variable **you must assign** the variable a value!

___

After declaring an immutable variable the value of the variable is fixed, meaning you cannot change or reassign the variable's value.

```js
const x = 15; // immutable variable declaration & assignment
x = 16; // not allowed
x += 1; // not allowed
console.log(x + 1); // allowed
```

---

### Block Scoping

> A variable's scope is determined by where the variable is declared.

___

```js
{ // scope 1
  // x does not exist here because it has not been declared
  let x; // declare variable in scope 1
  { // scope 2
    x = 5; // x exists here
  } // end scope 2
  console.log(x); // x exists here
} // end scope 1
console.log(x); // x DOES NOT exist here
```

`x` is declared in scope 1 and is usuable in scopes 1 and 2 after it has been declared.

However, `x` is not in scope outside of scopes 1 and 2, or before x has been declared.

### Array

> Arrays are JavaScript's list-like objects.

Arrays allow you to store any number of primitive literal values, variables, or objects as the **elements** of that array.

Arrays are indexed starting at 0.

___

#### Array Examples

```js
const someArray = ["a", 5, false];
```

someArray elements:

- 0: "a"
- 1: 5
- 2: false

```js
const someArray = ["a", 5, false];
console.log(someArray[0]); // prints "a"
console.log(someArray[1]); // prints 5
console.log(someArray[2]); // prints false
```

___

#### Array `length` Property

Arrays have a built in `length` property that will return the number of elements in the array.

```js
const someArray = ["a", 5, false];
console.log(someArray.length); // prints 3, the number of elements in someArray
```

___

#### Subarrays

An array can hold a different array as an element.

```js
const someArray = ["a", 5, false, [10, 9, 8]];
console.log(someArray[3]); // prints [10, 9, 8]
console.log(someArray[3][0]); // prints 10
console.log(someArray[3][1]); // prints 9
console.log(someArray[3][2]); // prints 8
```

### Non Iterative Array methods

- `.indexOf()`
- `.slice()`

___

#### `.indexOf()`

> Searches the array for a value, and returns the index of the first occurrence. If the value is not found it returns `-1`.

```js
const someArray = ["a", "b", "c"];
console.log(someArray.indexOf("a")); // prints 0
console.log(someArray.indexOf("c")); // prints 2
console.log(someArray.indexOf("dog")); // prints -1
```

___

#### `.slice`

> Takes an array and returns a substring of the array starting at a specific index.

```js
const someArray = ["a", "b", "c"];
console.log(someArray.slice(0)); // prints ["a", "b", "c"]
console.log(someArray.slice(1)); // prints ["b", "c"]
console.log(someArray.slice(2)); // prints ["c"]
console.log(someArray.slice(1, 2)); // prints ["b"]
```

---

### Object Literal

> Object literal's are one of JavaScript's key value pair data structures.

___

#### Object Literal Example

```js
const someObject = {"name": "Bernie", "age": 5};
console.log(someObject.name); // prints "Bernie"
console.log(someObject.age); // prints 5
```

___

#### Object Static Methods

> The Object class has a few static methods that provide information about a provided object.

```js
const someObject = {"name": "Bernie", "age": 5};
console.log(Object.keys(someObject)); // prints ["name", "age"]
console.log(Object.values(someObject)); // prints ["Bernie", 5]
```

### Conditional Logic

#### Equality Operators

- `==`: equality
- `!=`: inequality
- `===`: identity
- `!==`: nonidentity

___

equality & inequality examples:

```js
5 == 5 // true
5 == "5" // true
5 != 5 // false
5 != "5" // false
```

identity & nonidentity examples:

```js
5 === 5 // true
5 === "5" // false
5 !== 5 // false
5 !== "5" // false
```

___

#### Relational Operators

- `>`: greater than
- `<`: less than
- `>=`: greater than or equal to
- `<=`: less than or equal to

___

greater than & less than examples:

```js
5 > 5 // false
5 < 5 // false
5 > 4 // true
4 < 5 // true
```

greater than or equal to & less than or equal to examples:

```js
5 >= 5 // true
5 <= 5 // true
5 >= 4 // true
4 <= 5 // true
```

___

#### Logical Operators

- `&&`: and
- `||`: or

___

logical and examples:

```js
(5 > 8) && (5 > 6) // (false) && (false) -> false
(5 > 4) && (5 > 5) // (true) && (false) -> false
(5 > 9) && (5 > 4) // (false) && (true) -> false
(5 > 3) && (5 > 4) // (true) && (true) -> true
```

logical or examples:

```js
(5 > 8) || (5 > 6) // (false) || (false) -> false
(5 > 4) || (5 > 5) // (true) || (false) -> true
(5 > 9) || (5 > 4) // (false) || (true) -> true
(5 > 3) || (5 > 4) // (true) || (true) -> true
```

### Conditional Statements

- `if`
- `else`
- `else if`

___

#### `if`

```js
if (x > 5) { // if the value of x is greater than 5
  // do something
}
```

The code inside the if block is only executed if the condition is satisfied.

___

#### `else`

```js
if (x > 5) { // if the value of x is greater than 5
  // do something
}
else { // if the original if statement was not true
  // do something
}
```

If the original `if` statement is not satisfied the code inside the else block will be executed.

One of the branches **must** execute.

___

#### `else if`

```js
if (x > 5) { // if the value of x is greater than 5

}
else if (x < 2) { // if the value of x is less than 2 and the preceding if statement was false

}
```

If the original `if` statement is not satisfied the conditional statement in the following `else if` statement resolves.

If the `else if` condition is satisfied the `else if` block of code executes.

___

#### Chaining `else if` statements

You can add any number of additional `else if` statements to an existing `if` statement.

```js
if (a) { // if a resolves to true

}
else if (b) { // if b resolves to true and the preceding if statement was false

}
else if (c) { // if c resolves to true and the preceding if statement was false

}
```

`else if` chains always resolve in order, and only one code block can execute.

___

#### Extending `else if` with `else`

An `else` statement can be added after any number of `else if` statements.

```js
if (a) { // if a resolves to true

}
else if (b) { // if b resolves to true and the preceding if statement was false

}
else if (c) { // if c resolves to true and the preceding if statement was false

}
else { // if ALL preceding if statements were false

}
```

### Loops

- `while`
- `for`

---

### `while` loops

> Using the keyword `while` a code block will be executed until some condition is met.

Used when you do not know how many times a code block will need to be executed.

___

#### `while` example

```js
let x = 0;
while(x !== 5) {
  // code to be executed
  // x undergoes some change
}
```

---

### `for` loops

> Using the keyword `for` a code block will be executed a specific amount of times.

Used when you do know how many times a code block needs to be executed.

___

#### `for` example

```js
for (let x = 0; x < 100; x++) {
  // code to be executed exactly 100 times (0-99)
}
```

___

#### `for of`

> Code to be executed for each iterable in an **iterable object**.

`Strings` and `Arrays` are two examples of iterable objects.

___

##### `for of` examples

String example:

```js
const greeting = "hello";
for (const letter of greeting) {
  // code to be executed 
  // the variable: letter is available in this scope
  console.log(letter); // on iteration 1 this would print "h";
}
```

Array example:

```js
const names = ["Bernie", "Zelda", "Dutch"];
for (const name of names) {
  // code to be executed
  // the variable: name is available in this scope
  console.log(name); // on iteration 1 this would print "Bernie"
}
```

___

#### `for in`

> Code to be executed for each **enumerable property** of an object.

___

##### `for in` example

```js
const testObject = {"a": 15, "b": false, "c": "hello"};
for (const property in testObject) {
  // code to be executed
  // the variable: property is available in this scope
  console.log(property); // this would print one of "a", "b", or "c" on each iteration
  console.log(testObject.get(property)); // this would print one of 15, false, or "hello" on each iteration
}
```

JavaScript Objects have no memory of the order in which keys value pairs were inserted into the object.

This means you cannot predict the order of properties that will be looped through when using a `for in` loop.

#### `function` keyword

> In JavaScript a function can be declared by using the `function` keyword while providing the **function signature** and the **function body**.

```js
// function keyword and function signature
function someFunction() 
{ // <- start function body
  // some code to execute when someFunction is invoked
} // <- end function body
```

___

##### Invoking a Function

> A function is invoked by calling the name of the function followed by parenthesis. This is called the **function signature**.

```js
function greeting() { // <- function declaration
  console.log("Hello, World"); // <- function body
}

greeting(); // <- function invocation
```

---

#### Function Parameters & Invocation Arguments

> A function may define any number of parameters in its signature. These parameters define the values that are needed by the function.

```js
function greeting(name) { // <- the variable: name is the greeting functions sole parameter
  console.log("Hello " + name); // <- name is used in the function body
}

greeting("Paul"); // <- upon invoking the function we must provide an argument that will become the name parameter
```

output:

```js
Hello Paul
```

___

function parameter & argument example:

```js
function sumTwoNumbers(numberOne, numberTwo) {
  console.log(numberOne + numberTwo);
}

sumTwoNumbers(5, 15);
```

output:

```js
20
```

- **parameters** are defined by the signature and used in the function body.
- **arguments** are supplied when a function is invoked

---

#### Function Return Statements

> Functions can pass data out by using a return statement.

Data is passed into a function in the form of arguments, data can be passed out of a function with a return statement.

```js
function greeting(name) {
  return "Hello " + name; // returns the value "Hello " + name
}

const greetingMessage = greeting("Paul"); // the return value ("Hello Paul") is stored in the greetingMessage variable
```

___

function return example:

```js
function sumTwoNumbers(numberOne, numberTwo) {
  return numberOne + numberTwo;
}

const sum = sumTwoNumbers(5, 15);
console.log(sum);
```

output:

```js
20
```

- return statements give the ability to pass data **out** of a function
- arguments give the ability to pass data **in** to a function

---

#### JavaScript Arrow Functions

> In ES6 a new syntax was introduced for declaring functions called arrow functions.

You can recognize arrow functions by looking for the arrow syntax: `=>`.

___

arrow function example:

```js
const sumTwoNumbers = (numberOne, numberTwo) => {
  return numberOne + numberTwo;
}

const sum = sumTwoNumbers(5, 15);
console.log(sum);
```

output:

```js
20
```

We have eliminated the use of the `function` keyword, but not much else has changed.

We can continue to reduce the syntax using some additional tricks of arrow syntax.

___

additional reductions:

```js
const sumTwoNumbers = (numberOne, numberTwo) => numberOne + numberTwo;

const sum = sumTwoNumbers(5, 15);
console.log(sum);
```

output:

```js
20
```

We omit the curly brackets that normally wrap the function body and we omit the `return` statement.

In their place we substitute one expression thats resolution will implicitly return.

___

single parameter example:

```js
const greeting = name => "Hello " + name;
const greetingMessage = greeting("Paul");
console.log(greetingMessage);
```

output:

```js
Hello Paul
```

Functions with a single parameter are even shorter because you can omit the parenthesis around the sole parameter.

___

Functions with no parameters still need empty parenthesis:

```js
const helloWorld = () => console.log("Hello World");
helloWorld();
```

output:

```js
Hello World
```

---

#### Anonymous Functions

> In JavaScript you can declare anonymous functions, which are functions that do not have a name.

Anonymous functions cannot be invoked by the developer because they don't have a name.

Anonymous functions are used regularly with callbacks and you will learn more about them in a later topic.

___

##### Examples

using the `function` keyword:

```js
function() {
  return "Hello World";
}
```

using arrow syntax:

```js
() => "Hello World";
```

#### Type Coercion

> Type coercion is the implicit conversion between data types.

As a developer you may manually convert between data types, known simply as *type conversion*:

```js
const numberAsString = "10.5"
const number = parseFloat("10.5");

```

___

##### Type Coercion Example

In JavaScript sometimes the context requires a type conversion that is initiated by JavaScript and not manually by the developer. When this happens we refer to it as **type coercion**. For example:

```js
const someString = "Hello";
if (someString) {
  console.log(someString);
}
```

This is valid code in JavaScript. The variable someString with the value "Hello" must resolve to either true or false so the conditional statement can determine if it should execute or not. In this case the value "Hello" is considered a truthy value and so is converted to true.

___

#### Truthy/Falsy

> Due to type coercion **all** values must be resolved to a boolean representation of true or false when a boolean condition is encountered.

If a value resolves to true it is known as **truthy**. Most values are considered truthy.

If a value resolves to false it is known as **falsy**. The following slide contains all the falsy values.

___
