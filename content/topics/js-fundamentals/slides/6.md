### Invoking a Function

> A function is invoked by calling the name of the function followed by parenthesis. This is called the **function signature**.

<!-- CLARITY: incorrect definition of signature
MDN: https://developer.mozilla.org/en-US/docs/Glossary/Signature/Function
-->

```js
function greeting() { // <- function declaration
  console.log("Hello, World"); // <- function body
}

greeting(); // <- function invocation
```

## Function Parameters & Invocation Arguments

> A function may define any number of parameters in its signature. These parameters define the values that are needed by the function.

<!-- CLARITY: simplify wording
> A function may define any number of **ordered parameters**
-->

<!-- TODO: long comments dont render well (horizontal scrolling)

```js
function greeting(name) { // <- the variable: name is the greeting functions sole parameter
  console.log("Hello " + name); // <- name is used in the function body
}

greeting("Paul"); // <- upon invoking the function we must provide an argument that will become the name parameter
```

output:

```js
Hello Paul
```

___

function parameter & argument example:

```js
function sumTwoNumbers(numberOne, numberTwo) {
  console.log(numberOne + numberTwo);
}

sumTwoNumbers(5, 15);
```

output:

```js
20
```

- **parameters** are defined by the signature and used in the function body.
- **arguments** are supplied when a function is invoked

## Function Return Statements

> Functions can pass data out by using a the keyword `return`.

Data is passed into a function in the form of arguments, data can be passed out of a function with a return statement.

<!-- CLARITY: returning values vs expressions
is it worth showing an example first of purely returning a value 

then followup on "returning an expression will cause the expression to be evaluated before being returned"
-->

```js
function greeting(name) {
  return "Hello " + name; // returns the value "Hello " + name
}

const greetingMessage = greeting("Paul"); // the return value ("Hello Paul") is stored in the greetingMessage variable
```

___

> function return example:

```js
function sumTwoNumbers(numberOne, numberTwo) {
  return numberOne + numberTwo;
}

const sum = sumTwoNumbers(5, 15);
console.log(sum); // 20
```

<!-- ALIGNMENT: this is a rehash from above but also something we can expect of students based on prog funds edge -->

- return statements give the ability to pass data **out** of a function
- arguments give the ability to pass data **in** to a function

## Arrow Functions

> In ES6 a new syntax was introduced for declaring functions called arrow functions.

You can recognize arrow functions by looking for the arrow syntax: `=>`.

___

> arrow function example:

```js
const sumTwoNumbers = (numberOne, numberTwo) => {
  return numberOne + numberTwo;
}

const sum = sumTwoNumbers(5, 15);
console.log(sum); // 20
```

We have eliminated the use of the `function` keyword, but not much else has changed.

<!-- CLARITY: there is a difference between them beyond syntax

in favor of cutting it rather than going into the details (or do so in idiosyncrasies?)
-->

We can continue to reduce the syntax using some additional tricks of arrow syntax.

___

> shorthand syntax:

```js
const sumTwoNumbers = (numberOne, numberTwo) => numberOne + numberTwo;

const sum = sumTwoNumbers(5, 15);
console.log(sum); // 20
```

We omit the curly brackets that normally wrap the function body and we omit the `return` statement.

In their place we substitute one expression thats resolution will implicitly return.

<!-- CLARITY: lead with rules of shorthand syntax before examples

- single param: param =>

___

single parameter example:

```js
const greeting = name => "Hello " + name;
const greetingMessage = greeting("Paul");
console.log(greetingMessage); // Hello Paul
```

Functions with a single parameter are even shorter because you can omit the parenthesis around the sole parameter.

___

Functions with no parameters still need empty parenthesis:

```js
const helloWorld = () => "Hello World";
console.log(helloWorld()); // Hello World
```

## Anonymous Functions

<!-- CLARITY: see note above in introducing declaration vs expression -->

> In JavaScript you can declare anonymous functions, which are functions that do not have a name.

Anonymous functions cannot be invoked by the developer because they don't have a name.

Anonymous functions are used regularly with callbacks and you will learn more about them in a later topic.

___

> using the `function` keyword:

```js
function() {
  return "Hello World";
}
```

using arrow syntax:

```js
() => "Hello World";
```

In modern JavaScript (ES6+) arrow function syntax is most commonly used
